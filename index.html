EDGE

为什么选龙虾控制台

  • ⚡

    一条命令跑起来

    下载/克隆本仓库后执行 npm run start 即可拉起控制台（环境与前置依赖以 README 为准）。把「装半天、文档翻几轮」收敛成单一入口，专治安装难。

  • ◇

    本地真相 · 网关同源

    openclaw.json、agents、sessions 与 Gateway 实时对齐；健康轮询、流式重启与诊断直连本机进程，拒绝黑盒状态。

  • ◎

    多 Agent 一屏掌控

    专家战队、任务追踪、会话与通道同导航可达；像素办公室三套场景把「谁在干活」做成热力、榜单与可编辑布局。

  • ▸

    模型可实验、可落库

    单模型探测、临时 Key、内网预设，通过后写入配置；六语 + 五套主题，与生产面板同一套交互与持久化。

  • ⚙

    配置可视化、少手抄 JSON

    模型、密钥与通道尽量用向导与表单落盘，配合探测与校验再写回配置，降低配错环境与漏项的概率，缓解配置难。

  • ⊞

    状态集中，管理有抓手

    健康、重启、告警、会话与通道在同一套导航里串起来；哪里卡住、谁异常，不必在多窗口间猜，管理路径更短。

WHY ONE CLAW

四条设计原则

不是「再做一个聊天窗」，而是让 Agent 基础设施在屏幕上站得住、找得到、点得动。

  • 01

    文件即真相

    openclaw.json、agents、sessions 仍是权威；面板读写都围绕本机目录与网关。

  • 02

    运维可扫一眼

    Token、耗时、告警、通道与会话分布在多张表里，但共享同一套导航与主题。

  • 03

    模型改动可实验

    探测、临时 Key、内网预设，通过后再写配置，减少手抄 JSON 出错。

  • 04

    工作室带点趣味

    像素场景与热力、榜单是「状态可视化」，让值班氛围轻松一点。

INSTALL

按系统选型

与「桌面一键包」不同路径；这里描述的是本仓库面板的常见装法。

Windows

可选 NSIS 安装包（Electron 内嵌 Node，拉起 standalone Next）。开发者也可用 Node 18+ 直接 clone 跑 dev。

macOS

当前仓库主维护路径为源码 + npm；需本机 Node。可配合 Docker。

Linux

与 macOS 相同：Node 18+ 或 Docker；适合跑在 homelab / VPS 上仅内网访问。

Docker

仓库提供镜像说明时可整仓构建；挂载 OPENCLAW_HOME 与网关网络需自行规划。
